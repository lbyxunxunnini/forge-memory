# Forge Memory — 项目说明

## 项目背景

Forge Memory 是面向研发智能体的本地项目记忆层工具。

在大型项目中，智能体（如 Claude Code）每次执行任务时都需要从零理解项目结构、文件关系和历史变更，浪费大量 token 和时间。Forge Memory 的目标是：**先扫描项目生成结构化索引，再按任务生成聚焦的上下文包，让智能体精准读取源码、制定计划、执行和验证。**

```text
用户任务
  -> Forge Memory 生成项目上下文包
  -> task-driver 计划阶段消费上下文包
  -> 智能体精准读取源码、制定计划、执行和验证
```

### 设计原则

- **自动触发**：智能体需要项目上下文时自动调用，无需手动激活
- **静态优先**：不做服务器、不做后台进程，纯本地文件存储
- **零依赖**：仅使用 Python 标准库 + sqlite3
- **增量更新**：基于 content_hash 的增量扫描，二次扫描只更新变化文件
- **分支隔离**：多分支项目互不覆盖，每个分支独立索引

## 安装

### 环境要求

- Python 3.10+
- Git（用于分支检测和 commit 历史）

### 获取代码

```bash
git clone https://github.com/lbyxunxunnini/forge-memory.git
```

### 无需安装依赖

Forge Memory 仅使用 Python 标准库，无需 `pip install`。

## 使用说明

### CLI 命令

所有命令通过 `forge_memory.py` 统一入口调用：

```bash
# 初始化项目记忆
python3 forge_memory.py init <项目目录>

# 扫描项目（自动检测分支、生成索引、记录 commit 历史）
python3 forge_memory.py scan <项目目录>

# 查看扫描状态
python3 forge_memory.py status <项目目录>

# 生成任务级上下文包
python3 forge_memory.py context <项目目录> --task "任务描述"

# 文件影响分析
python3 forge_memory.py impact <项目目录> <文件路径>

# 导入索引到 SQLite
python3 forge_memory.py import-db <项目目录>

# 会话记忆管理
python3 forge_memory.py session add <项目目录> --title "标题"
python3 forge_memory.py session list <项目目录>
```

### 典型工作流

```bash
# 1. 首次使用：初始化 + 扫描
python3 forge_memory.py init ./my-project
python3 forge_memory.py scan ./my-project

# 2. 日常使用：查看状态
python3 forge_memory.py status ./my-project

# 3. 执行任务前：生成上下文包
python3 forge_memory.py context ./my-project --task "修改登录页面的表单验证"

# 4. 分析某个文件的影响范围
python3 forge_memory.py impact ./my-project lib/pages/login/login_page.dart

# 5. 会话结束时：保存会话摘要
python3 forge_memory.py session add ./my-project --title "登录功能重构"
```

### 生成的目录结构

```text
.project-context/
  context.json              # 项目元信息（含当前分支）
  project-summary.md        # 项目全貌摘要
  INDEX.md                  # 索引入口
  branches/
    <分支名>/
      index/
        files.jsonl         # 文件索引（path, role, language, content_hash, symbols, imports）
        modules.jsonl       # 模块索引
        commits.jsonl       # Git commit 历史（最近 50 个）
        commit_files.jsonl  # 每个 commit 涉及的文件
      graph/
        nodes.jsonl         # 知识图谱节点
        edges.jsonl         # 知识图谱关系
      scans/
        latest.json         # 最近一次扫描摘要
        history.jsonl       # 扫描历史
      packs/
        latest-context-pack.md  # 最近一次上下文包
      forge-memory.db       # SQLite 索引（可选）
  sessions/
    index.md                # 会话摘要索引
```

## 当前版本能力

### v0.1 基础扫描与上下文包

- 项目初始化与结构化扫描
- content_hash 增量更新
- JSONL 静态索引（files、modules）
- 知识图谱（nodes、edges）
- 任务级上下文包生成（按任务描述匹配相关文件）
- 中文→英文术语映射
- 会话记忆管理

### v0.2 分支隔离与 Git 记忆（已完成）

- 分支隔离存储：`.project-context/branches/<分支名>/`
- 旧结构自动迁移到当前分支子目录
- Git commit 历史记录（per-branch，追加式，最近 50 个）
- `impact` 命令：文件影响分析（导入关系、同模块文件、近期 commit、风险信号）
- 过期检测：impact 自动检测新 commit 并更新

### v0.3 SQLite 索引（已完成）

- JSONL → SQLite 导入（`import-db` 命令）
- SQLite 查询后端（文件查询、commit 查询）

## 可优化项

| 项目 | 当前状态 | 优化方向 |
|---|---|---|
| 依赖匹配精度 | 子串匹配，可能误匹配 | 改用完整路径匹配或 import 路径规范化 |
| commit_files 变更类型 | 全部标记为 `modified` | 解析 `git log --diff-filter` 区分 add/modify/delete |
| 大仓库性能 | 单线程扫描 | 可加并发扫描 |
| 符号提取覆盖 | 支持 Python/TS/Dart/Go/Swift | 扩展更多语言 |
| 模块推断 | 基于目录前缀启发式 | 可结合 package.json/pubspec.yaml 的依赖关系 |
| context pack 质量 | 基于关键词匹配 | 可加 TF-IDF 或轻量 embedding |
| SQLite 查询接口 | 只有基础查询 | 可加全文搜索、聚合查询 |
| 错误处理 | 基础 try-catch | 可加更细粒度的错误恢复 |
