# Forge Memory — 项目说明

## 项目背景

Forge Memory 是面向研发智能体的本地项目记忆层工具。

在大型项目中，智能体（如 Claude Code）每次执行任务时都需要从零理解项目结构、文件关系和历史变更，浪费大量 token 和时间。Forge Memory 的目标是：**先扫描项目生成结构化索引，再按任务生成聚焦的上下文包，让智能体精准读取源码、制定计划、执行和验证。**

```text
用户任务
  -> Forge Memory 生成项目上下文包
  -> 智能体精准读取源码、制定计划、执行和验证
```

### 设计原则

- **自动触发**：智能体需要项目上下文时自动调用，无需手动激活
- **静态优先**：不做服务器、不做后台进程，纯本地文件存储
- **零依赖**：仅使用 Python 标准库 + sqlite3
- **增量更新**：基于 content_hash 的增量扫描，二次扫描只更新变化文件
- **分支隔离**：多分支项目互不覆盖，每个分支独立索引

## 使用说明

### CLI 命令

所有命令通过 `forge_memory.py` 统一入口调用。完整命令列表和参数说明见 `SKILL.md` 的"脚本"章节，以下为常用命令速览：

```bash
# 初始化项目记忆
python3 forge_memory.py init <项目目录>

# 扫描项目（自动检测分支、生成索引、记录 commit 历史）
python3 forge_memory.py scan <项目目录>

# 查看扫描状态
python3 forge_memory.py status <项目目录>

# 生成任务级上下文包
python3 forge_memory.py context <项目目录> --task "任务描述"

# 文件影响分析
python3 forge_memory.py impact <项目目录> <文件路径>

# 导入索引到 SQLite
python3 forge_memory.py import-db <项目目录>

# 会话记忆管理
python3 forge_memory.py session add <项目目录> --title "标题"
python3 forge_memory.py session list <项目目录>
```

### 典型工作流

```bash
# 1. 首次使用：初始化 + 扫描
python3 forge_memory.py init ./my-project
# 输出：✓ 已创建 .project-context/ 目录

python3 forge_memory.py scan ./my-project
# 输出：✓ 扫描完成，索引 150 个文件，记录 50 个 commit

# 2. 日常使用：查看状态
python3 forge_memory.py status ./my-project
# 输出：
# 项目：my-project
# 分支：main
# 索引文件数：150
# 最近扫描：2026-07-07 10:00:00
# 最近 commit：2026-07-07 09:30:00

# 3. 执行任务前：生成上下文包
python3 forge_memory.py context ./my-project --task "修改登录页面的表单验证"
# 输出：
# # Forge Memory Context Pack
# 
# ## Task
# 修改登录页面的表单验证
# 
# ## Likely Entry Files
# - `lib/pages/login/login_page.dart` (source)
# - `lib/services/auth_service.dart` (source)
# 
# ## Related Modules
# - `login`
# - `auth`
# 
# ## Evidence And Caveats
# - 置信度：**high**
# - 匹配关键词：login, auth, form, validation
# - 相关文件数：8
# - 总文件数：150

# 4. 分析某个文件的影响范围
python3 forge_memory.py impact ./my-project lib/pages/login/login_page.dart
# 输出：
# # Impact Analysis: lib/pages/login/login_page.dart
# 
# ## Direct Imports
# - `lib/services/auth_service.dart`
# - `lib/widgets/form_field.dart`
# 
# ## Same Module Files
# - `lib/pages/login/login_controller.dart`
# - `lib/pages/login/login_state.dart`
# 
# ## Recent Changes
# - `abc1234` 2026-07-07 fix: login form validation
# - `def5678` 2026-07-06 feat: add login page
# 
# ## Risk Points
# - 高导入文件：`lib/services/auth_service.dart` (12 imports)

# 5. 会话结束时：保存会话摘要
python3 forge_memory.py session add ./my-project --title "登录功能重构"
# 输出：✓ 会话摘要已保存，ID: session-2026-07-07-001
```

### 生成的目录结构

```text
.project-context/
  context.json              # 项目元信息（含当前分支）
  project-summary.md        # 项目全貌摘要
  INDEX.md                  # 索引入口
  branches/
    <分支名>/
      index/
        files.jsonl         # 文件索引（path, role, language, content_hash, symbols, imports）
        modules.jsonl       # 模块索引
        commits.jsonl       # Git commit 历史（最近 50 个）
        commit_files.jsonl  # 每个 commit 涉及的文件
      graph/
        nodes.jsonl         # 知识图谱节点
        edges.jsonl         # 知识图谱关系
      scans/
        latest.json         # 最近一次扫描摘要
        history.jsonl       # 扫描历史
      packs/
        latest-context-pack.md  # 最近一次上下文包
      forge-memory.db       # SQLite 索引（可选）
  sessions/
    index.md                # 会话摘要索引
```

## 当前版本能力

### v0.1 基础扫描与上下文包

- 项目初始化与结构化扫描
- content_hash 增量更新
- JSONL 静态索引（files、modules）
- 知识图谱（nodes、edges）
- 任务级上下文包生成（按任务描述匹配相关文件）
- 中文→英文术语映射
- 会话记忆管理

### v0.2 分支隔离与 Git 记忆（已完成）

- 分支隔离存储：`.project-context/branches/<分支名>/`
- 旧结构自动迁移到当前分支子目录
- Git commit 历史记录（per-branch，追加式，最近 50 个）
- `impact` 命令：文件影响分析（导入关系、同模块文件、近期 commit、风险信号）
- 过期检测：impact 自动检测新 commit 并更新

### v0.3 SQLite 索引（已完成）

- JSONL → SQLite 导入（`import-db` 命令）
- SQLite 查询后端（文件查询、commit 查询）

### v0.4 TRAICE 评分提升（已完成）

- 错误信息规范化：`format_error` 函数 + 5 个异常类
- scan 降级策略：扫描前备份索引，失败时自动回滚
- import-db 重试机制：失败时自动重试 1 次
- 大项目分批扫描：MAX_BATCH_SIZE=1000，MAX_BATCH_BYTES=10MB
- 扫描状态持久化：`scan-progress.json`
- 置信度 reasons schema：基于规则的 `{overall, reasons[]}`
- 误用警告：检测项目太小、非 git 项目、临时目录等
- 一键初始化：`quickstart` 命令
- impact 匹配精度提升：路径前缀匹配
- CI 自动化验证：GitHub Actions
- 集成测试：端到端工作流测试
- 可观测性指标：文件覆盖率、索引新鲜度、Hash 覆盖率
- 状态检查命令：`doctor` 命令
- CLI 命令帮助：每个命令增加用法说明

## 常见错误 FAQ

### Q1: 运行 init 命令时报错 "已存在 .project-context 目录"
**原因**：项目已经初始化过，重复初始化会覆盖现有数据。
**解决**：
- 如果想重新初始化，使用 `--force` 参数：`python3 forge_memory.py init <项目目录> --force`
- 如果想保留现有数据，直接使用 `scan` 命令更新索引

### Q2: 运行 scan 命令时超时或中断
**原因**：项目文件过多，单次扫描耗时过长。
**解决**：
- 使用 `--max-files` 参数限制扫描文件数：`python3 forge_memory.py scan <项目目录> --max-files 1000`
- 使用 `--batch-size` 参数分批扫描：`python3 forge_memory.py scan <项目目录> --batch-size 500`

### Q3: 生成的上下文包不准确
**原因**：任务描述与项目索引的匹配度不足。
**解决**：
- 使用更具体的任务描述，包含关键词
- 使用 `--entry` 参数指定入口文件：`python3 forge_memory.py context <项目目录> --task "任务" --entry "path/to/file"`
- 先运行 `scan` 更新索引

### Q4: impact 命令找不到相关文件
**原因**：文件路径不正确或索引未更新。
**解决**：
- 检查文件路径是否正确（相对于项目根目录）
- 运行 `scan` 更新索引
- 使用 `status` 命令检查索引状态

### Q5: import-db 命令失败
**原因**：SQLite 数据库损坏或文件权限问题。
**解决**：
- 检查 `.project-context/branches/<分支名>/forge-memory.db` 文件是否存在
- 使用 `--backup` 参数从备份恢复：`python3 forge_memory.py import-db <项目目录> --backup`
- 删除数据库文件重新导入

### Q6: 非 git 项目无法使用
**原因**：forge-memory 依赖 git 进行分支检测和 commit 历史记录。
**解决**：
- 先初始化 git：`git init`
- 或使用 `--no-git` 参数（部分功能受限）：`python3 forge_memory.py scan <项目目录> --no-git`

### Q7: 项目太小（< 10 个文件）时扫描无意义
**原因**：小项目不需要复杂的索引和上下文包。
**解决**：
- 直接阅读项目文件，不需要使用 forge-memory
- 或使用 `--force` 参数强制扫描

### Q8: 扫描时包含敏感文件（.env、credentials）
**原因**：默认扫描会包含所有文件，可能泄露敏感信息。
**解决**：
- 使用 `.gitignore` 排除敏感文件
- 或使用 `--exclude` 参数手动排除：`python3 forge_memory.py scan <项目目录> --exclude ".env" "credentials*"`

### Q9: 会话摘要无法恢复
**原因**：会话文件损坏或路径不正确。
**解决**：
- 检查 `sessions/index.md` 文件是否存在
- 使用 `session list` 命令查看可用会话：`python3 forge_memory.py session list <项目目录>`
- 手动恢复：直接阅读 `sessions/*.md` 文件

### Q10: 大项目（> 10000 文件）扫描性能差
**原因**：单线程扫描，文件过多导致耗时过长。
**解决**：
- 使用 `--max-files` 参数限制扫描文件数
- 使用 `--batch-size` 参数分批扫描
- 使用 `--skip-tests` 参数跳过测试文件：`python3 forge_memory.py scan <项目目录> --skip-tests`
