# Forge Memory 评分提升计划

**当前评分**：4.5 / 5.0
**目标评分**：4.8 / 5.0
**评审日期**：2026-07-07
**评分框架**：TRAICE（Trust / Reliability / Adaptability / Convention / Effectiveness）

## 差距分析

| 维度 | 当前 | 目标 | 差距 | 关键缺失 |
|---|---|---|---|---|

| A · 适用性 | 4.5 | 4.8 | 0.3 | 触发方式优化 |
| C · 规范性 | 4.2 | 4.8 | 0.6 | 反模式与FAQ、文档质量、渐进式披露 |

## 逐项改进清单

### 1. R · 可靠性 4.5 → 4.8

| # | 改进项 | 具体做法 | 优先级 |
|---|---|---|---|
| 1.1 | 错误信息规范化 | 所有 CLI 错误输出统一格式：`[错误类型] 一句话说明 → 建议操作`，如 `[GitError] 未检测到 git 仓库 → 请先运行 git init` | P1 |
| 1.2 | scan 降级策略 | scan 中途失败时自动回滚到上一次成功状态，输出明确错误信息和恢复命令 | P1 |
| 1.3 | import-db 重试机制 | import-db 失败时自动重试 1 次（从 .db.bak 恢复），超过则输出人工干预指引 | P1 |
| 1.4 | 大项目分批扫描 | 超过 5000 文件时自动分批，每批完成后保存进度，中断后可从断点继续 | P1 |
| 1.5 | 扫描状态持久化 | 扫描过程中定期写入 `.project-context/scan-progress.json`，记录已完成的文件数和批次 | P1 |
| 1.6 | 置信度 reasons schema | context pack 输出的 confidence 从纯字符串改为 `{overall, reasons[]}`，说明为什么是 high/medium/low | P1 |
| 1.7 | 可观测性指标 | status 命令输出索引健康度：files.jsonl 行数 vs 项目实际文件数、commits.jsonl 最新时间 vs 当前时间差 | P2 |

### 2. A · 适用性 4.5 → 4.8

| # | 改进项 | 具体做法 | 优先级 |
|---|---|---|---|
| 2.1 | 自动触发判断优化 | 在 SKILL.md 中增加自动触发的判断流程图，明确什么情况下应该自动触发 | P1 |
| 2.2 | 触发条件示例 | 在 SKILL.md 中增加触发条件的正例和反例，每个类别至少 2 个 | P1 |

### 3. C · 规范性 4.2 → 4.8

| # | 改进项 | 具体做法 | 优先级 |
|---|---|---|---|
| 3.1 | SKILL.md 正例/反例 | 在触发纪律章节增加正例（什么情况触发）和反例（什么情况不触发），每个类别至少 2 个具体场景 | P0 |
| 3.2 | 边界条件说明 | 在 SKILL.md 中明确：空项目、大项目（10000+ 文件）、非 git 项目、detached HEAD、分支名含特殊字符等场景的行为 | P1 |
| 3.3 | 常见错误 FAQ | 在项目说明.md 或 SKILL.md 中增加 FAQ 章节，列出 5-10 个常见错误和解决方案 | P0 |
| 3.4 | 误用警告 | 在初始化和扫描时检测常见误用场景（如项目太小、非 git 项目），给出友好提示 | P1 |
| 3.5 | 完整操作示例 | 每个命令在项目说明.md 中至少有 1 个完整示例（含输入和预期输出截图或文本） | P0 |
| 3.6 | 端到端工作流示例 | 在项目说明.md 中增加一个完整的端到端示例：从初始化到生成上下文包，展示每一步的输出 | P1 |
| 3.7 | CLI 命令帮助 | 每个命令支持 `--help` 参数，输出用法说明和示例 | P2 |
| 3.8 | SKILL.md 精简 | 将详细流程拆分到 references/，SKILL.md 控制在 150 行内，只保留核心触发规则和状态选择 | P0 |
| 3.9 | 快速开始指南 | 在 README.md 或项目说明.md 中增加 "5 分钟快速开始" 章节，只讲最核心的 3 个命令 | P0 |
| 3.10 | 分层文档导航 | 在 SKILL.md 顶部增加文档导航，告诉用户 "想快速上手看这里，想深入理解看那里" | P1 |

### 4. E · 有效性 4.5 → 4.8

| # | 改进项 | 具体做法 | 优先级 |
|---|---|---|---|
| 4.1 | 一键初始化 | 支持 `forge_memory.py quickstart <project-root>` 命令，自动执行 init + scan + 生成示例上下文包 | P1 |
| 4.2 | impact 匹配精度提升 | 对同名不同模块的文件增加路径前缀匹配，减少误匹配 | P1 |
| 4.3 | CI 自动化验证 | 添加 GitHub Actions 配置，push 时自动运行 `python3 -m unittest discover` | P1 |
| 4.4 | 集成测试 | 添加端到端测试：在一个真实项目上运行 init→scan→status→context→impact→import-db 全流程 | P1 |
| 4.5 | 默认值优化 | scan 命令自动检测项目类型并设置合理的默认参数，减少用户需要指定的选项 | P2 |
| 4.6 | 状态检查命令 | 增加 `forge_memory.py doctor <project-root>` 命令，检查环境依赖和索引健康状态 | P2 |
